# 📄 BRD: Веб-приложение мотосервиса (Frontend Vue 3)

---

## 1. 📌 Описание продукта

Веб-приложение для сервиса, который занимается мототехникой:

- ремонт мотоциклов и другой мототехники (квадроциклы, снегоходы)
- поиск и подбор запчастей
- мотоподбор по Грузии
- мотоэвакуатор (специально оборудованный пикап для перевозки мотоциклов)
- заказ мотоциклов с аукционов США и Японии

Также включены услуги:

- полное оформление
- растаможка
- фото и видео отчёты по процессу доставки

---

## 2. 🎯 Цель продукта

- дать пользователю возможность заказать услуги
- для клиентов **заказа мотоцикла с зарубежного аукциона** — обеспечить прозрачное отслеживание доставки, фото/видео отчёты и уведомления по этому заказу
- для остальных услуг — приём заявки и дальнейшая коммуникация вне сценария «кабинет по коду» (в рамках текущего BRD фронта)

---

## 3. 👤 Тип пользователя

### Клиент (общий)

- оставляет заявку на интересующую услугу
- по заявке с ним связывается сервис (канал связи вне рамок этого документа или уточняется отдельно)

### Клиент услуги «заказ / подбор мотоцикла с аукциона за границей»

- после **фактической оплаты** этой услуги получает **уникальный код доступа** к отслеживанию не из формы заявки на сайте, а **в Telegram от бота**
- код выдаётся ботом **только после того**, как **администратор** полностью занесёт в систему информацию о мотоцикле и **даст отмашку боту** на отправку кода клиенту
- по этому коду открывает **личный кабинет отслеживания** (только этот сценарий использует раздел «отслеживание по коду» на фронте)

---

## 4. 🧾 Основные функции системы

---

## 4.1 Услуги

Система предоставляет следующие услуги:

- ремонт мототехники
- поиск запчастей
- подбор мотоциклов по Грузии
- мотоэвакуатор
- заказ мотоциклов с аукционов США и Японии

---

## 4.2 Заявка

Пользователь может оставить заявку на любую услугу.

После отправки заявки (любая услуга):

- заявка регистрируется для обработки сервисом
- **уникальный код для входа в кабинет отслеживания не выдаётся автоматически** и **не связан** с этой формой для услуг, отличных от аукционного импорта

Отдельный процесс **только для услуги заказа с зарубежного аукциона** (оплата → данные о мотоцикле у админа → команда боту → код в Telegram) описан в разделах **5** и **7**.

---

## 5. 🔐 Личный кабинет отслеживания (по коду)

**Область применения:** раздел «отслеживание заказа по коду» на сайте предназначен **исключительно** для клиентов услуги **подбора / заказа мотоцикла с аукциона за границей** (США, Япония и т.д.), которые получили код **из Telegram-бота** после оплаты и действий администратора.

Для заявок на ремонт, запчасти, подбор по Грузии, эвакуатор и т.п. **этот кабинет не используется** (нет выдачи кода для входа в данный сценарий).

Доступ осуществляется по **уникальному коду** (без пароля), введённому в интерфейсе отслеживания.

---

### В личном кабинете отслеживания аукционного заказа отображается:

---

## 5.1 Информация о заказе

- текущий статус заказа
- дата прибытия (если известна)
- тип услуги

---

## 5.2 Медиа

- фотографии мотоцикла
- видео отчёты

Контент обновляется по мере появления новой информации.

---

## 5.3 Прогресс доставки

Отображается прогресс-бар, который показывает этап доставки:

- процесс обработки / покупки
- транспортировка
- таможня
- прибытие в Грузию
- выдача владельцу

Также показывается:

- примерное время до получения мотоцикла

---

## 5.4 История обновлений

Пользователь видит список всех изменений:

- добавлены фото
- изменён статус
- обновлён этап доставки
- добавлены видео

---

## 6. 🔔 Уведомления

Для **заказа с зарубежного аукциона** при изменениях по отслеживаемому заказу:

- пользователь получает уведомление в **Telegram** (в т.ч. первичная выдача кода ботом по сценарию из раздела 7)
- уведомления включают:
  - новый статус
  - новые фото/видео
  - важные изменения по заказу

---

## 7. 🔑 Уникальный код доступа (аукцион за границей)

Цепочка выдачи кода (не через публичную форму заявки на сайте):

1. Клиент оплачивает услугу подбора / заказа с зарубежного аукциона.
2. Администратор заполняет в системе (админка / бэкенд — вне фронт-BRD) полную информацию о мотоцикле по этому заказу.
3. Администратор даёт **отмашку Telegram-боту** на отправку клиенту уникального кода.
4. **Telegram-бот** отправляет клиенту **уникальный код доступа**.
5. Клиент вводит код на сайте в разделе отслеживания и попадает в личный кабинет по этому аукционному заказу.

Свойства кода:

- код используется для входа в **личный кабинет отслеживания** указанного аукционного заказа
- код является **единственным** способом доступа к информации по этому заказу для клиента на фронте (без пароля)

## 9. 📱 Требования к интерфейсу

---

### Общие требования:

- простой интерфейс
- понятное разделение: **заявка на услуги** vs **отслеживание аукционного заказа по коду из Telegram**
- для аукционного заказа — быстрый доступ по коду, понятный статус и этапы
- адаптация под мобильные устройства

---

### Личный кабинет отслеживания (по коду, только аукцион за границей):

- отображение текущего статуса
- отображение прогресса доставки
- список фото и видео
- история изменений

---

## 10. 🚀 MVP (первая версия)

В MVP входит:

- форма заявки на услуги (все типы из раздела 4.1)
- раздел ввода кода и **личный кабинет отслеживания только для заказа с зарубежного аукциона** (после получения кода из Telegram по процессу из раздела 7)
- отображение статуса, прогресса доставки, фото и видео отчётов, истории обновлений — **в этом кабинете**
- интеграция с **Telegram-ботом**: выдача кода клиенту и уведомления об изменениях по аукционному заказу

Исключено из описания MVP как обязательное для всех заявок:

- автоматическая выдача кода доступа сразу после отправки общей формы заявки на сайте

---

## 11. 🧠 Ключевая идея продукта

Для клиента **аукционного заказа** пользователь в кабинете по коду всегда должен понимать:

- где находится его мотоцикл
- что с ним происходит
- какие изменения были сделаны
- когда он его получит
